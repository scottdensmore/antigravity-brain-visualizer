   rerun the verifier after addressing it.

   diff and all staged, unstaged, and untracked files. Address every actionable
   finding before committing.
8. After verification and code review are complete, commit using Conventional
   Commits format.
9. Before creating a pull request:
   - Confirm that local verification remains valid.

     unstaged, or untracked content.
   - Do not repeat code review when the already-reviewed diff and worktree are
     unchanged.
